# Canva-Style Design Platform Overhaul

## Overview
Build a professional design platform like Canva.in with a modern UI/UX, high-fidelity template library, and drag-and-drop editor functionality.

## Architecture
- **New Editor Route**: `/editor` (replaces current FabricEditor in future)
- **Existing Editor**: `/fabric-editor` (kept as fallback)
- **Template Source**: Local `template-library.json` for quick loading

### 2.3 Layered Template Composition
Each template JSON must include:

1. **Background Layer** (z-index: 0)
   - Unsplash image with `backgroundImage` property
   - Or solid color/gradient

2. **Overlay Layer** (z-index: 1-10)
   - Semi-transparent shapes (opacity: 0.1-0.5)
   - Decorative circles, rectangles, lines

3. **Typography Layer** (z-index: 11+)
   - Main Heading: 48-80px, bold
   - Subheading: 24-32px
   - Body text: 14-20px

### Phase 4: Template Loading/Saving Flow

**4.1 Template Selection Flow**
1. User clicks template in sidebar
2. Fetch canvas JSON from `/api/templates/:id/canvas`
3. Clear canvas: `canvas.clear()`
4. Set dimensions: `canvas.setDimensions({w, h})`
5. Load: `canvas.loadFromJSON(canvasData, callback)`

**4.2 Save as Template Flow**
1. Admin clicks "Save as Template"
2. Export canvas: `canvas.toJSON()`
3. Generate thumbnail: `canvas.toDataURL()`
4. Upload thumbnail to Cloudinary
5. POST to `/api/templates/save` with metadata
