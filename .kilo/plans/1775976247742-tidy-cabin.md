# Template-to-Editor Workflow Implementation Plan

## Project Overview
Implement a Canva-like template-to-editor workflow using Fabric.js for JSON-based canvas editing, with a categorized template gallery and live modification capabilities.

## Current State Analysis

### Existing Codebase
- **Template Data**: `client/src/data/editableTemplates.js` - Contains ~50+ templates with `zones` array (editable text positions)
- **Template Gallery**: `client/src/pages/TemplatesPage.jsx` - Displays template grid with category filtering
- **Editor**: `client/src/pages/CustomizePoster.jsx` - Uses react-draggable + HTML rendering, exports via html2canvas
- **Export**: Uses html2canvas (flat image) instead of JSON serialization

### Gap Analysis
1. Templates use custom zone-based format, not Fabric.js JSON format
2. No image placeholder support in current template structure
3. Export loses editability (PNG only, not JSON)
4. No Fabric.js canvas - uses HTML DOM elements
5. No shape/vector element support

---

## Implementation Plan

### Phase 1: JSON Schema Design

**New Template Schema (Fabric.js compatible)**
```json
{
  "templateId": "instagram-post-1",
  "name": "Travel Post",
  "category": "Social Media",
  "subcategory": "Instagram Post",
  "canvas": {
    "width": 1080,
    "height": 1080,
    "background": "#1a1a2e"
  },
  "objects": [
    { "type": "text", "text": "Paris", "left": 540, "top": 200, "fontFamily": "Poppins", "fontSize": 48, "fill": "#ffffff" },
    { "type": "image", "src": "placeholder-mountains.jpg", "left": 540, "top": 400, "width": 800, "height": 400 },
    { "type": "rect", "left": 100, "top": 100, "width": 200, "height": 200, "fill": "#ff6b6b" }
  ]
}
```

**Template Categories**
- Social Media (Instagram Post, Facebook Cover, Twitter Post, LinkedIn Banner)
- Business (Resume, Presentation, Flyer, Logo)
- Marketing (Ad Banner, Poster, Brochure)
- Personal (Birthday Card, Invitation, Certificate)

---

### Phase 2: Database/API Changes

**Server Routes**
- `GET /api/templates` - List all templates with filtering
- `GET /api/templates/:id` - Get single template JSON
- `POST /api/templates` - Create new template
- `PUT /api/templates/:id` - Update template

**MongoDB Model (Template.js)**
```javascript
{
  templateId: String,
  name: String,
  category: String,
  subcategory: String,
  canvas: { width: Number, height: Number, background: String },
  objects: [{
    type: String, // text, image, rect, circle, triangle, path
    // Fabric.js properties
  }],
  thumbnail: String,
  isPublic: Boolean,
  createdAt: Date
}
```

---

### Phase 3: Template Gallery UI Enhancement

**Category Sidebar Component**
- Collapsible category sections
- Category icons + counts
- Active category highlighting
- Search within categories

**Template Card**
- Thumbnail preview (rendered canvas)
- Template name
- Category badge
- "Use Template" button

---

### Phase 4: Fabric.js Editor Integration

**New Editor Components**
1. `CanvasEditor.jsx` - Fabric.js canvas wrapper
2. `Toolbar.jsx` - Add text, shapes, images, upload
3. `PropertiesPanel.jsx` - Edit selected object properties
4. `LayerPanel.jsx` - Reorder/manage objects

**Fabric.js Features**
- Load template from JSON (`canvas.loadFromJSON`)
- Text editing with double-click
- Image replacement via upload
- Drag/drop positioning
- Resize handles
- Rotation
- Z-index layering
- Undo/Redo (canvas history)

**Export Options**
- PNG (`canvas.toDataURL`)
- JSON (save template for re-editing)
- PDF (via jsPDF)

---

### Phase 5: Live Modification Features

**Text Editing**
- Select text object → Properties panel shows textarea
- Live preview as typing
- Font family, size, color, weight, alignment

**Image Replacement**
- Select image object → "Replace Image" button
- Upload from device or stock images
- Maintains aspect ratio option

**Element Manipulation**
- Drag to reposition
- Corner handles to resize
- Rotation handle
- Delete key to remove
- Duplicate (Ctrl+D)

---

### Phase 6: Migration Strategy

**Keep both systems during transition**
1. Create new Fabric.js template format
2. Add `isFabricTemplate` flag to distinguish
3. Gradually migrate existing templates
4. Run both editors in parallel

## Acceptance Criteria

1. **Template Gallery**: Grid displays categorized templates with filtering
2. **JSON Storage**: Templates stored as Fabric.js-compatible JSON
3. **Editor Loading**: Click template → Fabric.js canvas loads with all objects
4. **Text Editing**: Double-click text → edit inline, properties panel updates JSON
5. **Image Replacement**: Select image → upload new → updates src in JSON
6. **Drag & Drop**: All objects draggable within canvas bounds
7. **Export**: Download as PNG, Save as JSON (re-editable), Export PDF
8. **Categories**: Filter by Social Media, Business, Marketing, Personal
9. **Undo/Redo**: Canvas state history for editing operations
